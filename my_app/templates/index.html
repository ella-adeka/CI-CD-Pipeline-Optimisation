Project Lifecycle

This project focuses on automating the continuous integration and continuous deployment (CI/CD) of a web application using Jenkins and Docker. The goal is to streamline the development process, reduce manual intervention, and ensure efficient and reliable application delivery.

The activities of the pipeline include:

  1. Code: A simple web application was created with the Python Framework 'Flask'. VSCode was the code editor of choice.
  2. Build: Jenkins was configured to automatically trigger builds upon pull requests made to the repository.
  3. Test: Unit and integration tests were conducted for the development environment to ensure individual components function as intended. End-to-End (E2E) tests were performed in the staging environment, simulating real-world scenarios.
  4. Release: Jenkins was used to automate deployment to the staging environment. Chnages were promoted to the production environment follwoing successful testing in the staging environment.
  5. Monitor: Promoetheus and Grafana were implemented to collect and store metrics from the application and infrastructure. Continuous monitoring will provide real-time insights into application health, performance, and resource utilisation.
